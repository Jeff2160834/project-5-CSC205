Throughout project 5 I learned a lot about the three algorithms that we implemented and how they work.
Starting with Gnome sort the easiest one of all to start with. 
This is a very simple sorting algorithm that is based on the idea of a gnome sorting a line of flower pots. The gnome looks at the current pot and the next one, if they are in the correct order he moves to the next pot, if they are not he swaps them and moves back one pot. 
This process continues until the gnome has gone through the entire line of pots. 
This algorithm is not very efficient and is not used in practice, 
but it was a good starting point for understanding sorting algorithms.
The AI for the most part was able to successfully implement this algorithm on the first try using the pseudocode that we were given.

The next algorithm that we implemented was the cocktail sort. This is a variation of bubble sort that goes in both directions on each pass through the list.
This algorithm is more efficient than bubble sort in some cases, but it is still not very efficient and is not used in practice. 
The AI was able to implement this algorithm on the first try with not many mistakes, but it did have some issues with the implementation of the while loop and the use of the swapped variable.
This was a good learning experience for me to see how the AI can struggle with certain aspects of programming and seeing how I need to manually fix it or improve my prompts.

The last algorithm that we implemented was the shell sort algorithm. This is a more efficient sorting algorithm that is based on the idea of insertion sort, but it allows for the exchange of items that are far apart.
This algorithm is more efficient than insertion sort and is used in practice. The way shell sort works is by first sorting elements that are far apart from each other and then progressively reducing the gap between the elements to be sorted.
That was the fundamental idea that I gathered when attempting to understand the algorithm and how it works.
